**Lifecycle and detach.** Container exits when you close the terminal; new `-d` flag for background runs.

**Run any command in a disposable container. Optionally run an action on the result.**

---

## What's new

- **Terminal close = container down** — When you run dockpipe attached (default), closing the terminal or disconnecting the client now tears down the container. The entrypoint traps SIGTERM/SIGHUP and stops the inner command; we use Docker’s `--init` so the container has a proper PID 1 that forwards signals. No more lingering containers when you close the window.
- **`-d` / `--detach`** — Run the container in the background. Use when you want to start a long-running run (e.g. Claude session) and disconnect. `docker run -d` is used; the command prints the container ID. Use `docker logs <id>` or `docker attach <id>` as needed. When the command inside exits, the container is removed (`--rm`).

## Installation (Linux)

```bash
wget https://github.com/jamie-steele/dockpipe/releases/download/v0.4.2/dockpipe_0.4.2_all.deb
sudo dpkg -i dockpipe_0.4.2_all.deb
```

Requires Bash and Docker. If `dpkg` reports missing deps: `sudo apt install -f`.

---

## Upgrade from 0.4.1

Replace the .deb and reinstall as above. All usage unchanged; new behavior is default (attach = exit on terminal close) plus optional `-d`.

---

## Examples

**Attached (default): close terminal → container exits**

```bash
dockpipe --template agent-dev -- claude -p "review this"
```

**Detached: run in background, get container ID**

```bash
dockpipe -d --template agent-dev -- claude -p "review this"
# docker logs <container_id>  or  docker attach <container_id>
```
