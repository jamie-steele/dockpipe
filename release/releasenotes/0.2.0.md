Documentation, examples, and a few conveniences based on early feedback.

**Run, isolate, and act — pipe commands into disposable containers and act on the results.**

---

## What’s new

### Template: `agent-dev`

- Added **`--template agent-dev`** as the command-agnostic name for the Node + Claude Code image. Use it for AI/agent workflows without implying a single vendor.
- **`--template claude`** remains supported as an alias (same image). Both resolve to `dockpipe-claude`.

### Documentation

- **Permissions (UID/GID):** The runner has always used `-u "$(id -u):$(id -g)"` so files created in the container are owned by you. This is now documented in [docs/architecture.md](docs/architecture.md) and mentioned in the README.
- **Passing state between chained runs:** New section in the architecture doc: shared workdir as the main bridge, stdout → stdin for streaming, and an optional convention (e.g. `.dockpipe/result.json`) for structured data. Env vars do not persist across separate `dockpipe` invocations.
- **When the primitive isn’t enough:** New section on escape hatches—orchestration, rich multi-step state, image limits—and how to compose dockpipe with Makefiles, shell, or other tools instead of stretching the core.
- **AGENTS.md:** Added “Limitations and escape hatches” and a note to prefer `agent-dev` in docs for broader appeal.

### Full chained examples

- **examples/chained-non-ai/** — Runnable lint → test → build (Makefile), plus generate → validate (scripts). Each step runs in a fresh container with a shared workdir.
- **examples/chained-multi-ai/** — Plan → implement → review with Claude: copy-paste commands for a real repo or the included `project/` dir, with a shared workdir and optional commit action.

### Other

- README “Full examples” section now links to chained-non-ai, chained-multi-ai, and the Claude worktree example.
- Blog post linked from the repo: [Run, Isolate, and Act (DEV Community)](https://dev.to/jamie-steele/run-isolate-and-act-a-minimal-primitive-for-container-workflows-553m).

## Example (unchanged)

```bash
# Generic
dockpipe -- ls -la

# With agent-dev (or claude)
echo "refactor the auth module" \
  | dockpipe --template agent-dev --action examples/actions/commit-worktree.sh \
  -- claude --dangerously-skip-permissions -p "$(cat)"
```

## Installation (Linux)

```bash
wget https://github.com/jamie-steele/dockpipe/releases/download/v0.2.0/dockpipe_0.2.0_all.deb
sudo apt install ./dockpipe_0.2.0_all.deb
```

Requires Bash and Docker. If `apt` reports missing deps: `sudo apt install -f`.

---

## Upgrade from 0.1.0

- Replace the .deb and reinstall as above. All 0.1.0 usage remains valid; `--template claude` still works. Prefer `--template agent-dev` in new scripts or docs if you want the generic name.
