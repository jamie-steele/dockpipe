**Default data volume.** Persistent agent environment by default; named volume or bind mount. You only need to mount your work directory.

**Run any command in a disposable container. Optionally run an action on the result.**

---

## What's new

- **Default data volume** — A named volume `dockpipe-data` is mounted at `/dockpipe-data` and set as `HOME`. Tool state (e.g. first-time Claude login) and repos persist there; same volume each run. You only mount your work dir (default: cwd).
- **`--data-vol <name>`** — Use a different named volume (Docker terminology). Default is `dockpipe-data`.
- **`--data-dir <path>`** — Bind mount a host path for persistent data instead of a volume (e.g. `$HOME/.dockpipe`). Use when you want repos or state on the host (e.g. for cherry-pick from a worktree).
- **`--no-data`** — Do not mount the data volume (minimal run; no persistence).

Terminology matches Docker: **named volume** vs **bind mount**.

## Installation (Linux)

```bash
wget https://github.com/jamie-steele/dockpipe/releases/download/v0.5.0/dockpipe_0.5.0_all.deb
sudo dpkg -i dockpipe_0.5.0_all.deb
```

Requires Bash and Docker. If `dpkg` reports missing deps: `sudo apt install -f`.

---

## Upgrade from 0.4.x

Replace the .deb and reinstall as above. New default: data volume is mounted and `HOME=/dockpipe-data`. Use `--no-data` if you want no data mount.

---

## Examples

**Default: named volume (reusable agent environment)**

```bash
dockpipe --template agent-dev -- claude -p "review this"
# First run: log in. Next run: same volume, no re-login.
```

**Bind mount a host path**

```bash
dockpipe --data-dir "$HOME/.dockpipe" --template agent-dev -- claude -p "review this"
```

**Minimal run (no data volume)**

```bash
dockpipe --no-data -- ls -la
```
