First public release.

**Run, isolate, and act — pipe commands into disposable containers and act on the results.**

---

## What is dockpipe?

dockpipe is a minimal CLI built around one primitive:

**spawn → run → act**

1. Spawn a disposable container from an image.
2. Run your command or script inside it (your directory is mounted at `/work`).
3. Optionally run an action on the result (e.g. commit, export patch).

It’s composable and command-agnostic: the same flow works for any tool, script, or AI assistant.

---

## Features

- **Disposable container execution** — Run commands in isolated environments; no pollution of your host.
- **Pipe-friendly** — Stdin and stdout work naturally (e.g. pipe a prompt in, pipe output out).
- **Composable** — Use with your own scripts and actions; chain with other tools.
- **Command-agnostic** — Works with any CLI, script, or binary. AI tools (Claude, Codex, etc.) are one use case, not the only one.
- **Lightweight** — Bash and Docker only; no extra runtime dependencies.

---

## Example

Pipe in a task, run Claude in a container, commit the result:

```bash
echo "refactor the auth module" \
  | dockpipe --template claude --action examples/actions/commit-worktree.sh \
  -- claude --dangerously-skip-permissions -p "$(cat)"
```

Generic command (default image):

```bash
dockpipe -- ls -la
dockpipe -- bash -c "npm test"
```

## Installation (Linux)

```bash
wget https://github.com/jamie-steele/dockpipe/releases/download/v0.1.0/dockpipe_0.1.0_all.deb
sudo apt install ./dockpipe_0.1.0_all.deb
```

Requires Bash and Docker (`docker.io` or `docker-ce`). If `apt` reports missing deps: `sudo apt install -f`.

## Philosophy

dockpipe is intentionally minimal. It does not manage workflows, enforce structure, or provide orchestration. It provides a small execution primitive you compose with existing tools.
