**`dockpipe action init`** — scaffold an action script so the “Act” part is one command away.

**Run, isolate, and act — pipe commands into disposable containers and act on the results.**

---

## What’s new

### `dockpipe action init` / `dockpipe action create`

- **`dockpipe action init my-action.sh`** (or **`action create`**) writes a boilerplate action script in the current directory (or the path you give). No need to dig into `examples/` to see how actions work.
- The template includes:
  - Comments for **`DOCKPIPE_EXIT_CODE`** and **`DOCKPIPE_CONTAINER_WORKDIR`**
  - A simple success/failure branch and `exit` with the command’s code
  - `chmod +x` so you can use it right away with `--action my-action.sh`
- If you pass a path like `scripts/notify.sh`, the directory is created if missing.
- Refuses to overwrite an existing file.

**Example:**

```bash
dockpipe action init my-action.sh
# Created: /path/to/my-action.sh
# Use: dockpipe --action my-action.sh -- <your-command>

# Edit my-action.sh, then:
dockpipe --action my-action.sh -- make test
```

## Installation (Linux)

```bash
wget https://github.com/jamie-steele/dockpipe/releases/download/v0.3.0/dockpipe_0.3.0_all.deb
sudo apt install ./dockpipe_0.3.0_all.deb
```

Requires Bash and Docker. If `apt` reports missing deps: `sudo apt install -f`.

---

## Upgrade from 0.2.x

Replace the .deb and reinstall as above. All existing usage is unchanged.
