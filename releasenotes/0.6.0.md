**0.6.0 — Resolvers, worktree on host, commit on host, template init.**

**Run → Isolate → Act.** One primitive for any command; AI is one use case.

---

## What's new

- **Resolvers** — Agnostic AI provider support. One file per tool in `resolvers/` (e.g. `claude`, `codex`). Use `--resolver claude` or `--resolver codex`; add your own by dropping a file in `resolvers/` (no core changes). No vendor lock-in.
- **Worktree on host** — `--repo <url>` and `--branch <name>`: dockpipe creates the clone and worktree on the **host** (e.g. under `~/.dockpipe` or `--data-dir`), mounts it as `/work`, runs your command in the container. Same flow for Claude, Codex, or any resolver.
- **Commit on host** — When you use the bundled commit-worktree action (or `--repo`/`--branch`), the **commit runs on the host** after the container exits. The AI never has git access; your host identity and safety.
- **Template init** — `dockpipe template init my-ai [--from llm-worktree]` copies the workflow (config, resolvers, isolate). Run `dockpipe --workflow my-ai --repo <url> -- ...`; config points to scripts/. No run script in templates.
- **Commit on host uses your git config** — When the action runs on the host (current dir or `--repo`/`--branch`), the commit uses your existing host git identity; no need to pass git env vars.

## Installation (Linux)

```bash
wget https://github.com/jamie-steele/dockpipe/releases/download/v0.6.0/dockpipe_0.6.0_all.deb
sudo dpkg -i dockpipe_0.6.0_all.deb
```

Requires Bash and Docker. If `dpkg` reports missing deps: `sudo apt install -f`.

---

## Upgrade from 0.5.x

Replace the .deb and reinstall as above. New: `--resolver`, `--repo`, `--branch`, `template init`. Commit-worktree now runs on the host when you use the bundled action. Rebuild template images if you use versioned tags.

## Examples

**AI in worktree, commit on host (one primitive for any resolver):**

```bash
dockpipe --resolver claude --repo https://github.com/you/repo.git --branch claude/task \
  -- claude --dangerously-skip-permissions -p "Fix the bug"
```

**Copy a workflow template and customize (no upstream contribution needed):**

```bash
dockpipe template init my-ai --from llm-worktree
dockpipe --workflow my-ai --repo https://github.com/you/repo.git -- claude -p "Fix the bug"
```
