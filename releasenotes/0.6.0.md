# DockPipe v0.6.0

This is the first release we treat as **fully usable**: you can install into a project, run **workflows** in isolation (Docker by default), and rely on the CLI for day-to-day work. DockPipe **dogfoods itself in CI**—the same binary runs bundled workflows in automation.

---

## Highlights

- **DockPipe runs DockPipe** — CI exercises real **`--workflow`** flows against the shipping binary.
- **Architecture is settled** — **workflow** (what happens), **runtime** (where execution runs), **resolver** (which tool or platform), **strategy** (lifecycle wrappers), **assets** (shared support files under **`core/assets/`**).
- **Bundled layout is consistent** — embedded content unpacks to **`dockpipe/core/`** and **`dockpipe/workflows/`** (see below).
- **`dockpipe init` is the project entry point** — scaffolds scripts, images, **`templates/`**, and merged **`templates/core/`** (runtimes, resolvers, strategies, assets).
- **Bundled workflows** — e.g. **`run`**, **`run-apply`**, **`run-apply-validate`**, **`init`**, plus **`dockpipe/workflows/`** content in the materialized bundle (this repo also runs **`test`**, **`test-demo`**, and optional Codex flows in CI — see **AGENTS.md**).
- **Isolation-first execution** — disposable containers, project at **`/work`**, host bash for prep and follow-up when you need it.
- **Small surface, fast path** — one primitive (run → isolate → act); workflows add structure without turning the core into a heavy orchestrator.

---

## Bundled layout (materialized)

After the binary unpacks to your cache, the framework tree looks like:

```text
dockpipe/
  core/
    runtimes/
    resolvers/
    strategies/
    assets/
  workflows/
```

Authoring checkouts of this repo use **`templates/`** and **`templates/core/`** instead; the runner accepts both layouts.

---

## Usage

```bash
dockpipe init
dockpipe --workflow test --runtime docker
```

You need **Docker** and **bash** on the host. **`dockpipe doctor`** checks the toolchain and bundled assets.

---

## Notes

- **No MSI for this release** — install from **`.deb`**, **tarball**, **zip**, or the **Windows install script** (see **[docs/install.md](https://github.com/jamie-steele/dockpipe/blob/v0.6.0/docs/install.md)**).
- **Install is manual** — pick an artifact from [Releases](https://github.com/jamie-steele/dockpipe/releases), or build from source with **`make`** / **`go build`**.
- **Pre-1.0** — flags, env names, and YAML fields may still adjust slightly between releases; pin versions in production if you need stability.
