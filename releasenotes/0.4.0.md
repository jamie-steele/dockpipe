# dockpipe v0.4.0

**README and positioning refresh** — clarity-first for new users. No new features; same CLI, better first impression.

**Run any command in a disposable container. Optionally run an action on the result.**

---

## What's new

### README rewritten for 15-second clarity

- **Hero:** One-line value prop and explicit "general-purpose, not an AI framework; AI is one use case."
- **Try it (15 seconds):** Install and first run (`dockpipe -- make test`) in one block so visitors can go from zero to running in under a minute.
- **What you can do:** Table of four use cases with one command each (tests, script+commit, pipe stdin, AI+commit) plus `dockpipe action init`.
- **How it works:** Spawn → run → act in three bullets, placed after "what you can do" so benefit comes before mechanism.
- **Why not docker run:** Side-by-side feel—show the raw `docker run` one-liner you'd otherwise write, then what dockpipe adds (action phase, templates, pipe-friendly CLI, UID/GID).
- **Install:** Single table by platform (Linux .deb, macOS source, Windows/WSL note). Requirements: Bash, Docker.
- **Examples:** Consolidated into short blocks; full runnable examples linked. Templates and Actions sections tightened.

### Positioning

- General-purpose first: tests, scripts, codegen, or AI—same flow. AI is a strong use case but not the lead.
- First example is always generic (`dockpipe -- make test`); AI examples follow.

## Installation (Linux)

```bash
wget https://github.com/jamie-steele/dockpipe/releases/download/v0.4.0/dockpipe_0.4.0_all.deb
sudo apt install ./dockpipe_0.4.0_all.deb
```

Requires Bash and Docker. If `apt` reports missing deps: `sudo apt install -f`.

---

## Upgrade from 0.3.x

Replace the .deb and reinstall as above. All usage is unchanged; only docs and README changed.
