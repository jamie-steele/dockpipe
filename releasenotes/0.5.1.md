**Startup polish.** ASCII art banner and launch spinner when running attached in a TTY.

**Run any command in a disposable container. Optionally run an action on the result.**

---

## What's new

- **Banner** — When stdout is a TTY, dockpipe shows a block-letter “dockpipe” logo and “Run → Isolate → Act” before launching. No banner when piped or in scripts.
- **Launch spinner** — A short “Launching container…” line with a rotating `|/-\` spinner runs for about 0.6s, then is cleared before the container starts. All on stderr so it doesn’t mix with container output.
- **`action init --from`** — Copy a bundled action as a starting point: `dockpipe action init my-commit.sh --from commit-worktree` (or `export-patch`, `print-summary`). Edit the copy and use it with `--action my-commit.sh`.

---

## Installation (Linux)

```bash
wget https://github.com/jamie-steele/dockpipe/releases/download/v0.5.1/dockpipe_0.5.1_all.deb
sudo dpkg -i dockpipe_0.5.1_all.deb
```

Requires Bash and Docker. If `dpkg` reports missing deps: `sudo apt install -f`.

---

## Upgrade from 0.5.0

Replace the .deb and reinstall as above. Behavior unchanged except for the new TTY-only banner and spinner.

## Examples

**Attached in a terminal (you’ll see the banner and spinner)**

```bash
dockpipe --template agent-dev -- claude -p "review this"
```

**Resume a previous Claude session** (data volume keeps state)

```bash
dockpipe --template agent-dev -- claude --resume <session-id> --dangerously-skip-permissions
```

**Clone a bundled action to customize**

```bash
dockpipe action init my-commit.sh --from commit-worktree
# Edit my-commit.sh, then: dockpipe --action my-commit.sh -- ...
```

**Piped or non-TTY: no banner, no spinner**

```bash
dockpipe -- ls -la | cat
```
