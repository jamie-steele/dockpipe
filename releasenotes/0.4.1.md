# dockpipe v0.4.1

**Hotfix release.** Bug fix and small consistency fix.

**Run any command in a disposable container. Optionally run an action on the result.**

---

## What's fixed

- **.deb install:** `DOCKPIPE_REPO_ROOT` was derived from `dirname(BASH_SOURCE[0])/..`, which for `/usr/bin/dockpipe` (symlink) resolved to `/usr` instead of `/usr/lib/dockpipe`, so `runner.sh` was not found. The CLI now uses `/usr/lib/dockpipe` when run from `/usr/bin`, so the package works after `dpkg -i` with no extra config. ([#1](https://github.com/jamie-steele/dockpipe/issues/1))
- **Help text:** `--help` example now uses `--template agent-dev` instead of `claude` for consistency.

---

## Installation (Linux)

```bash
wget https://github.com/jamie-steele/dockpipe/releases/download/v0.4.1/dockpipe_0.4.1_all.deb
sudo dpkg -i dockpipe_0.4.1_all.deb
```

Requires Bash and Docker. If `dpkg` reports missing deps: `sudo apt install -f`.

**Note:** Prefer `dpkg -i` for a local .deb. Using `apt install ./file.deb` from your home directory can trigger a sandbox warning (apt’s `_apt` user can’t read the file); that’s an apt limitation, not a package bug. `dpkg -i` avoids it.

---

## Upgrade from 0.4.0

Replace the .deb and reinstall as above. All usage unchanged.
